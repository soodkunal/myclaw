# Code Style Rule

This rule should be copied into your `CLAUDE.md` to enforce general code style.

```markdown
## Algemene Code Stijl
1. **Clarity over Cleverness**: Write readable code. Avoid 1-line complex nested comprehensions or ternaries.
2. **Variable Naming**: Use descriptive names (`userAccountBalance` instead of `uab`).
3. **Comments**: Do NOT comment obvious code. Only comment the *Why*, not the *What*.
4. **Error Handling**: Do not swallow errors. Always log them appropriately and return standard error responses.
```
