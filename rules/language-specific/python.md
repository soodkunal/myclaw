# Python Language Rule

This rule should be copied into your `CLAUDE.md` when working in a Python codebase.

```markdown
## Python Rules
1. **Typing**: Use strong type hints everywhere (PEP 484).
2. **Docstrings**: Use Google-style docstrings for all classes and functions.
3. **Formatting**: Ensure all code is formatted using `black` and passes `ruff` checks.
4. **Dependencies**: Add all new packages to `requirements.txt` or `pyproject.toml` immediately.
```
